# Flask Application

This is a Flask-based web application that can be started using `python run.py`. It uses a virtual environment for dependency management.

## Prerequisites

Before running the application, ensure you have the following installed:
- Python (>=3.8 recommended)
- pip (Python package manager)
- virtualenv (for creating isolated Python environments)

## Setup

Follow these steps to set up and run the application:

### 1. Clone the Repository
```bash
git clone <repository-url>
cd <repository-name>
```

### 2. Create a Virtual Environment
```bash
python -m venv env
```

### 3. Activate the Virtual Environment
- **Windows:**
  ```bash
  env\Scripts\activate
  ```
- **Mac/Linux:**
  ```bash
  source env/bin/activate
  ```

### 4. Install Dependencies
```bash
pip install -r requirements.txt
```

### 5. Run the Application
```bash
python run.py
```

By default, the Flask application runs on `http://127.0.0.1:5000/`.

## Project Structure
```
├── env/                # Virtual environment (not included in version control)
├── static/             # Static files (CSS, JS, images)
├── templates/          # HTML templates
├── app.py              # Main Flask application file
├── run.py              # Entry point for running the Flask app
├── requirements.txt    # Dependencies
├── config.py           # Configuration settings
├── README.md           # Documentation (this file)
```

## Environment Variables
To configure the application, create a `.env` file in the project root and define necessary environment variables.

Example:
```
FLASK_APP=run.py
FLASK_ENV=development
SECRET_KEY=your_secret_key
```

## API Endpoints
| Method | Endpoint     | Description        |
|--------|-------------|--------------------|
| GET    | `/`         | Home Page         |
| GET    | `/status`   | Health Check      |

## Deployment
To deploy the application, use a production-ready server like **Gunicorn**:
```bash
pip install gunicorn
```
Run with:
```bash
gunicorn -w 4 run:app
```
